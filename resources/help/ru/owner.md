# Руководство по работе с системой hovera

> Добро пожаловать в hovera. Это руководство проведёт вас по всем функциям панели владельца конюшни. Большинство шагов начинаются с основной навигации в панели администратора `/app`.

---

## 1. Первые шаги

После входа в систему вы попадаете на панель `/app`. Слева расположена навигация, разделённая на группы:

- **Конюшня** — лошади, клиенты, денники, здания, специалисты, абонементы, уход, тарифы постоя
- **Календарь** — план дня, бронирования, регулярные занятия, инструкторы, манежи
- **Финансы** — счета
- **Настройки** — настройки конюшни, выставление счетов, платежи, KSeF, сотрудники

Для начала рекомендуем:

1. **Настройки конюшни** — заполните данные компании, брендинг, рабочие часы
2. **Здания** — создайте хотя бы одно («Главная конюшня»)
3. **Денники** — привяжите каждый к зданию
4. **Тарифы постоя** — определите услуги (сено, уборка денника, транспорт)
5. **Инструкторы + Манежи** — чтобы календарь мог принимать бронирования

---

## 2. Настройки конюшни

**Путь:** `/app/tenant-settings`

Разделы формы:

- **Идентификация** — название, юридическое название, ИНН
- **Локализация** — страна, язык по умолчанию, часовой пояс, валюта
- **Брендинг** — основной цвет, логотип, hero image (на публичной странице)
- **Публичная страница `/s/{slug}`** — слоган, часы работы, описание, контакты, соцсети
- **Онлайн-бронирование** — могут ли клиенты бронировать через публичную страницу; длительность урока, рабочие часы, минимальный срок

**Пример:**

| Поле | Значение |
|---|---|
| Название | Конюшня «Под Липами» |
| Слоган | Постой, уроки, отдых с 1995 года |
| Часы работы | Пн–Пт 9:00–20:00 · Сб–Вс 8:00–18:00 |
| Мин. срок | 4 часа |
| Макс. горизонт | 30 дней |

## 3. Лошади

**Путь:** `/app/horses`

### Добавление лошади

Нажмите **«Создать лошадь»** → заполните:

- **Кличка** (например, «Буцефал»)
- **Владелец** (выбор из списка клиентов; пусто = владелец — конюшня)
- **Денник** (выбор из активных денников)
- **Микрочип / Номер паспорта / UELN** (Universal Equine Life Number)
- **Пол:** Кобыла / Мерин / Жеребец / Племенной жеребец
- **Порода, масть, дата рождения**
- **Постой — начисляемые услуги** (множественный выбор из тарифа)

После сохранения лошадь появляется в списке. Столбцы: Кличка, Порода, Пол, Масть, Возраст, Владелец.

### Карточка лошади (вкладки)

- **Лошади → редактировать** открывает редактирование + 4 вкладки:
  - **Уход и здоровье** — прививки, ковка, стоматолог (с авто-подсказкой next_due)
  - **Активности** — кормление, чистка, выгул в леваду
  - **Сообщения** — чат с владельцем (mail-out)
  - **Документы** — паспорт, договор, страховка (PDF/JPG, до 25 МБ)

## 4. Денники и здания

### Здания — `/app/buildings`

1 конюшня (как место) может иметь несколько зданий: «Красная конюшня», «Новая конюшня», «Левадный павильон». Каждое здание — это группа денников.

**Поля:** название, порядок, активно.

### Денники — `/app/boxes`

Поля формы: здание, название/номер, краткий код (например, «12»), тип (внутренний / левада / уличный / карантин), размер (м²), месячная цена постоя, активен.

Столбцы таблицы:
- **Здание** (с группировкой — можно свернуть «Красную конюшню»)
- **Название, Тип, м²**
- **Статус** — Свободен (зелёный) / Занят (серый)
- **Пол лошади** (если назначена)
- **Постой (руб/мес)**

**Пример структуры:**

```
Здание «Красная конюшня»
  ├ Денник 1 — внутренний — Свободен
  ├ Денник 2 — внутренний — Занят (Кобыла «Луция»)
  └ Денник 3 — внутренний — Занят (Мерин «Буцефал»)

Здание «Восточная левада»
  ├ Лев 1 — левада — Свободен
  └ Лев 2 — левада — Занят (Племенной жеребец «Титан»)
```

## 5. Клиенты

**Путь:** `/app/clients`

### Добавление клиента

- **Тип:** частное лицо / семья / компания
- **Имя и фамилия / название**
- **Email, телефон**
- **ИНН** (с опцией «Получить из GUS», если настроен GUS API)
- **Идентификация владельца лошади (ARMiR):**
  - **№ EP** — номер производителя, присвоенный ARMiR при регистрации лошади в Центральной базе непарнокопытных
  - **PESEL** — fallback, если у владельца нет EP

### Карточка клиента

При входе в `Редактировать` вы видите:

- Все поля формы
- **Вкладка «Лошади»** — список лошадей, принадлежащих клиенту
- Действие **«Скопировать ссылку портала»** — генерирует magic link (TTL 30 мин) для копирования
- Действие **«Отправить ссылку на e-mail»** — отправляет email со ссылкой для входа

## 6. Календарь

### План дня — `/app/calendar`

Вид дня с группировкой по инструктору/манежу. Кликните пустой слот, чтобы добавить бронирование. Кликните запись, чтобы редактировать/удалить.

### Бронирования — `/app/calendar-entries`

Полная таблица бронирований. Фильтры: тип, статус, лошадь, инструктор, «только предстоящие».

**Статусы:** Заявлено (ожидает подтверждения) → Подтверждено → Завершено / Отменено / Неявка.

**Типы бронирования:** Индивидуальная езда, Групповая езда, Тренировка, Уход (вет/ковка), Событие, Блокировка.

### Регулярные занятия — `/app/recurring-calendar-entries`

Создаёт серию (например, «Группа пн. 17:00, еженедельно, до конца года»). Действие **«Сгенерировать вхождения»** раскладывает серию в отдельные бронирования (макс. 365 за раз).

---

## 7. Специалисты (ковали + ветеринары)

**Путь:** `/app/specialists`

Список внешних контрагентов и сотрудников конюшни, которые подковывают / лечат лошадей.

### Добавление

- **Специальность:** Ветеринар / Ковал
- Имя и фамилия, email, телефон, цвет в календаре
- **Аккаунт в системе (опционально)** — если специалист является сотрудником (TenantMembership), привяжите его к User. Тогда залогиненный сотрудник увидит представление **«Мои задачи»** со списком своих процедур.

### Связь с записями о здоровье

При создании HR (`/app/health-records`) выберите тип:
- «Ковал» → список специалистов фильтруется до `farrier`
- другие типы → список фильтруется до `vet`

В таблице HR столбец «Выполнил» показывает имя специалиста вместо пустого поля.

---

## 8. Сотрудники конюшни

**Путь:** `/app/team-members` (виден только для owner / admin)

### Добавление сотрудника

Нажмите **«Добавить сотрудника»** → введите:
- email, имя и фамилия, роль

Система автоматически:
- если пользователь существует → создаёт `TenantMembership`
- если не существует → отправляет `UserInvitation` (активационная ссылка письмом, TTL 7 дней)

### Роли

| Роль | Описание |
|---|---|
| **Владелец** | Полный доступ |
| **Админ** | То же, минус удаление конюшни |
| **Менеджер** | Управление календарём, счетами |
| **Инструктор** | Календарь, бронирования, свои лошади |
| **Сотрудник** | Activity log (кормление, уборка) |
| **Ветеринар** | Связь со специалистом + Мои задачи |
| **Только просмотр** | Read-only |

### Сброс пароля

Действие **«Отправить ссылку для сброса пароля»** (значок ключа рядом с активным сотрудником) — отправляет email со ссылкой на `/app/password-reset/request`. Срок действия — 60 минут.

## 9. Счета

### Конфигурация — `/app/invoicing-settings`

- **Нумерация FV / Proforma / Корректировка** — шаблон с плейсхолдерами `{seq}`, `{YYYY}`, `{MM}`, `{prefix}`
- **Сброс нумерации** — Ежегодно / Ежемесячно / Никогда
- **Срок оплаты по умолчанию** — например, 7 дней
- **Данные продавца** (snapshot на счёте)

### Выставление — `/app/invoices`

1. Нажмите **«Создать»** → выберите клиента (авто-заполнение данных покупателя)
2. Добавьте позиции (Repeater): название, количество, ед., цена нетто, НДС
3. Save → статус `Draft` → нажмите **«Выставить»** → номер и дата
4. Нажмите **«Отправить на e-mail»** → клиент получает ссылку на публичный вид (с опцией «Оплатить сейчас», если у вас активна онлайн-касса)
5. Нажмите **«Отправить в KSeF»** (если KSeF настроен) → подпись + отправка

### Корректировки

Нажмите **«Корректировка»** на проведённом счёте → создаёт черновик корректировки с нулевыми позициями; заполните количества «после изменения» → выставите.

---

## 10. Онлайн-платежи — `/app/payment-settings`

Выберите шлюз по умолчанию:

- **Нет** — всё офлайн (перевод, наличные)
- **Przelewy24** — введите Merchant ID + POS ID + CRC + API key
- **PayU** — POS ID + OAuth credentials
- **Stripe** — secret_key + webhook_secret
- **Mollie** — API key

После настройки каждый выставленный счёт имеет в письме кнопку **«Оплатить сейчас»**.

---

## 11. KSeF (электронные счета) — `/app/ksef-settings`

Требуется:
- **ИНН конюшни** (контекст KSeF)
- **Окружение:** test / demo / prod
- **Сертификат** — PFX (.pfx) или PEM (.crt + .key); ключ и пароль шифруются Laravel Crypt + AES-256

После загрузки выставленный счёт получает действие **«Отправить в KSeF»** — аутентификация сертификатом + отправка XML.

---

## 12. Публичная страница и embed-виджеты

### Публичная страница — `https://app.hovera.app/s/{slug}`

Рендерится автоматически на основе раздела «Публичная страница» из `/app/tenant-settings`. Основной цвет, логотип, hero image, описание, часы работы, контакты, соцсети. Опционально: список инструкторов, тарифы постоя, доступность денников.

### Embed-виджеты

В `/app/tenant-settings` → раздел «Виджеты» — готовые iframe-ы для вставки в Wordpress / Squarespace / собственный сайт:

- **Свободные денники** («У нас X свободных денников»)
- **Забронировать онлайн** (CTA к booking flow)
- **Тарифы постоя** (таблица)
- **Список инструкторов**

---

## 13. Портал клиента

Клиент входит через `https://app.hovera.app/s/{slug}/portal/login`:
- вводит email
- получает magic link письмом (TTL 30 мин)
- клик → попадает на дашборд

С уровня **`/app/clients/{id}`** вы можете сгенерировать ссылку вручную (для SMS/Messenger) или отправить письмом (действие «Отправить ссылку на e-mail»).

В портале клиент видит:
- предстоящие бронирования (с опцией «Перенести» / «Отменить»)
- свои абонементы (X / Y осталось)
- историю бронирований
- неоплаченные счета
- своих лошадей (с оповещениями о здоровье)
- сообщения

## 14. Дашборд «Сегодня» — `/app`

После входа сверху вы видите 4 плитки:
- 🗓️ **Бронирования сегодня** — клик ведёт в календарь,
- 🟢 **Свободные денники** — клик к списку денников,
- 🔴 **Просроченные процедуры** — клик к фильтру в уходе и здоровье,
- 💸 **Неоплаченные FV** — сумма руб + count, клик к фильтру «Выставленные».

Под ними — таблица сегодняшних активных бронирований (время · лошадь · инструктор · манеж · статус).

---

## 15. Шаблоны процедур — `/app/treatment-templates`

Предопределённые preset'ы для записей ухода — один клик заполняет тип, описание и предлагаемый срок следующего визита на основе `interval_days`.

Каждый новый tenant получает 6 стандартных: прививка столбняк/грипп (раз в год), EHV (раз в полгода), дегельминтизация (3 мес.), ковка (6 нед.), стоматолог (раз в год), ветконтроль (раз в полгода). Можно редактировать, деактивировать или добавить свои.

В форме новой записи в **Уход и здоровье** появляется select «Шаблон процедуры» — выбор авто-заполняет type/summary/details/next_due_at.

---

## 16. Карта веса лошади — вкладка в карточке лошади

Вписывайте измерения раз в месяц (вес в кг + опционально обхват груди). Столбец «Изменение» сравнивает с предыдущим измерением: 🟢 прирост, 🟡 убыль, ⚪ стабильно (разница < 5 кг).

---

## 17. План кормления — вкладка в карточке лошади

CRUD: время суток (утро/полдень/вечер/ночь), корм, количество, заметки. **Клиент видит план в портале** в разделе «План кормления» карточки лошади (read-only).

---

## 18. Склад кормов — `/app/feed-inventory`

Список кормовых позиций с остатком (`SUM(delta)`) и порогом оповещения. Действие **«+ Складское движение»** добавляет запись (приход / выдача / корректировка / списание); знаки движения вычисляются автоматически. Карточка позиции имеет вкладку **История движений** (audit log). Позиции ниже порога отображаются с числовым badge'м в сайдбаре.

---

## 19. Галерея фото лошади — вкладка в карточке лошади

Загрузка JPG/PNG/WEBP/HEIC до 10 МБ. Клиент видит грид в портале (плитки 1:1, lightbox). Отдельно от документов (паспорта / страховки остаются во вкладке «Документы»).

---

## 20. Экспорт `.ics` инструктора

В **Конюшня → Инструкторы** действие **«Календарь .ics»** в строке → модальное окно с URL фида. Вставьте в Google Calendar / Outlook / Apple как «Добавить календарь по URL». Уроки синхронизируются каждые несколько часов. Окно: 6 мес. назад + 12 вперёд. Отменённые уроки = `STATUS:CANCELLED`.

---

## 21. Отчёты — группа навигации «Отчёты»

4 ежемесячные страницы (с пикером месяца):

- **Доходы** — суммы нетто по корзинам: постой / уроки / абонементы / прочее + top 10 позиций.
- **Возрастная структура задолженности** — неоплаченные после срока, сгруппированные: 0–30 / 31–60 / 61–90 / 90+ дней с градиентом цветов.
- **Использование лошади** — count уроков на лошадь + сумма часов. >25 = риск перегрузки.
- **Использование инструктора** — часы + посещаемость % + cancelled + no-show.

---

## 22. Bulk invoicing — `/app/bulk-invoicing`

Массовая генерация FV Draft за месяц для всех клиентов на основе активных услуг постоя их лошадей (`Daily × дней месяца`, `Monthly × quantity`).

1. Выберите месяц (по умолчанию: предыдущий),
2. Проверьте предпросмотр (per-client список позиций + суммы),
3. Отметьте клиентов или оставьте всех,
4. **Сгенерировать черновики** → создаёт Draft Invoice per client.

Абонементы пропускаются (имеют auto-FV при продаже). Каждый Draft вы утверждаете отдельно в **Счета → Выставить** (номер + KSeF + mail).

---

## 23. Self-booking клиента в портале

Клиент после входа кликает «+ Забронировать урок» → выбирает свою лошадь, инструктора, день + слот из доступных → submit. В панели конюшни бронирование появляется со статусом **Ожидает** + `metadata.source = client_portal`. Подтверждаете так же, как другие бронирования.

## 24. Что видит какая роль — матрица доступа

Сайдбар и страницы **фильтруются по роли сотрудника**. Вет не увидит счета и абонементы, сотрудник не увидит настройки конюшни — минимизированный вид = меньший риск ошибки и более ясный интерфейс.

| Раздел | owner | admin | manager | trener | сотрудник | vet | viewer |
|---|:-:|:-:|:-:|:-:|:-:|:-:|:-:|
| Лошади · Уход и здоровье · План дня · Бронирования | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| Клиенты | ✓ | ✓ | ✓ | ✓ | — | — | ✓ |
| Денники · Тарифы постоя · Здания · Регулярные · Инструкторы · Манежи | ✓ | ✓ | ✓ | ✓ | — | — | ✓ |
| Специалисты · Шаблоны процедур | ✓ | ✓ | ✓ | — | — | ✓ | ✓ |
| Склад кормов | ✓ | ✓ | ✓ | — | ✓ | — | ✓ |
| Счета · Абонементы · Отчёты | ✓ | ✓ | ✓ | — | — | — | ✓ |
| Bulk invoicing | ✓ | ✓ | ✓ | — | — | — | — |
| Настройки (конюшни · FV · KSeF · Платежи) · Сотрудники | ✓ | ✓ | — | — | — | — | — |
| Мои задачи | — | — | — | — | — | ✓ | — |
| Помощь | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |

> **Master admin** (внешний саппорт hovera) видит всё независимо от роли — используется для impersonacji в целях отладки.

> **Смена роли** = изменение видимости сразу после обновления страницы. Можно переключить роль сотрудника в **Настройки → Сотрудники**.
